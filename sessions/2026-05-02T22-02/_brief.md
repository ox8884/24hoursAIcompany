# 📋 작업 브리프

**원 명령:** 그래서 내가 웹사이트를 보려면 내가 뭘 제공해주면 돼?

## 요약
사용자에게 웹사이트 확인을 위한 기술적 개념(API, 데이터 구조) 대신, 비즈니스 관점의 '정보 제공' 중심으로 답변해야 합니다. 현재 단계는 Mock-up 완성 후 '실제 기능 구현 및 테스트 케이스 정의' 단계이므로, 필요한 것은 원본 자료와 작동 방식에 대한 가이드라인입니다.

## 분배
- **📱 Secretary**: 기술 용어(API, 스캐폴딩)를 모두 배제하고, 사용자에게 '무엇을 제공해야 하는지'를 3단계의 간결한 체크리스트 형태로 설명하는 보고서 초안을 작성해 주세요. (예: 데이터 원본 자료, 목표 기능 정의, 주요 의사결정 시나리오)
- **🔍 Researcher**: 프로젝트의 핵심 주제(VA 혜택 기반 카페 창업)와 관련된 '현재 시장 트렌드 데이터' 또는 '필수 고려 법률/재정 지표' 3가지를 리스트업하고, 이를 웹사이트에 채워 넣어야 할 구체적인 숫자로 제시해 주세요. (예: Celina 지역 디저트 카페 평균 임대료 변화 추이, 특정 VA 지원금액의 최근 변동율 등)
- **💻 Developer**: 기술적 관점에서 '데이터 입력'을 요청받았으니, 개발자가 가장 먼저 확정해야 할 핵심 인터랙션 로직(Interaction Logic) 2가지를 정의해 주세요. (예: '지원금액 변경 시 예상 수익 변화 그래프 자동 업데이트', 'A 항목 클릭 시 관련 법규 팝업 표시') 이 로직은 단순한 데이터 연결을 넘어, 사용자가 스스로 의사결정을 하도록 유도하는 흐름이어야 합니다.
